# Installation Guide

## 📌 Introduction

This guide explains how to prepare your environment for studying and testing the Camphish project in an authorized lab or educational setting.

> **Note:** Use this project only in environments where you have explicit permission to perform security testing.

---

# System Requirements

Before getting started, make sure your system meets the following requirements:

* Linux (Recommended: Kali Linux)
* Git
* PHP
* Internet connection
* Terminal access

---

# Clone the Repository

Clone the repository to your local machine.

```bash
git clone https://github.com/techchipnet/CamPhish.git
```

Move into the project directory.

```bash
cd Camphish
```

---

# Verify PHP Installation

Check whether PHP is installed.

```bash
php --version
```

If PHP is installed correctly, the version information will be displayed.

---

# Verify Git Installation

Check whether Git is installed.

```bash
git --version
```

---

# Project Files

After cloning the repository, verify that the project files are available.

Example:

```text
README.md
LICENSE
templates/
assets/
```

The exact file structure may vary depending on the project version.

---

# Network Requirements

Some demonstrations may require network connectivity in a controlled lab environment. Ensure your firewall and network configuration allow the required connections.

---

# Browser Requirements

Use a modern web browser such as:

* Google Chrome
* Microsoft Edge
* Mozilla Firefox

These browsers provide support for modern web APIs and permission prompts.

---

# Keeping the Project Updated

If the repository receives updates, you can pull the latest changes.

```bash
git pull
```

---

# Troubleshooting

### Repository cannot be cloned

Possible causes:

* Incorrect repository URL
* Internet connectivity issues
* Git is not installed

---

### PHP is not recognized

Possible causes:

* PHP is not installed.
* PHP is not available in your system PATH.

---

### Permission Issues

If you encounter file permission problems, verify that your user account has appropriate access to the project directory.

---

# Best Practices

* Keep your system updated.
* Use the latest stable version of PHP.
* Work only in authorized lab environments.
* Read the project documentation before testing.

---

# Summary

Your environment is now ready for exploring the project in an educational and authorized setting. Continue with **Usage.md** to learn about the project workflow and its educational purpose.
